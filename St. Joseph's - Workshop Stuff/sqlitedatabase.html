SQLite

That's how.

Implementing SQLite in an app is very simple. Unlike, for example, Java, you don't have to modify or setup JDBC/ODBC or anything. SQLite is straight forward and a walk on a cake.

SQLite is an Open Source database. SQLite supports standard relational database features like SQL syntax, transactions and prepared statements. The database requires limited memory at runtime (approx. 250 KByte) which makes it a good candidate from being embedded into other runtimes, in our case, Android.

If your application creates a database, this database is by default saved in the directory
DATA/data/APP_NAME/databases/FILENAME

The Package to be imported:
android.database.sqlite

Steps in using SQLite:

1. Create Database

2. Create Table

3. Insert Rows

4. Update/ Retrieve/ Delete Rows

5. Close database connection

1. Creating a Database

Create an object.
SQLiteDatabase db;

db=openDatabase(String path, SQLiteDatabase.CursorFactory factory, int flags, DatabaseErrorHandler errorHandler)

db=openDatabase(String path, SQLiteDatabase.CursorFactory factory, int flags)

db=openOrCreateDatabase(String path, SQLiteDatabase.CursorFactory factory)

db=openOrCreateDatabase(File file, SQLiteDatabase.CursorFactory factory)

db=openOrCreateDatabase(Path, Flags, CursorFactory);

Some Flags used with db creation

OPEN_READWRITE

OPEN_READONLY

CREATE_IF_NECESSARY

2. Creating a Table

db.execSQL("CREATE TABLE STATEMENT");


Data types supported by SQLite

TEXT

INTEGER

REAL

NUMBER

And of course, NULL

3. Inserting Rows

db.execSQL("INSERT INTO tableName VALUES(...);");

4. Updating Rows

db.execSQL("UPDATE tableName SET(...);");

5. Deleting Rows

db.execSQL("DELETE FROM tableName WHERE...;");

5. Retrieving Rows

...

db.execSQL("SELECT ... FROM tableName WHERE ... ORDER BY ... ;"); ??

NOPE. That's not right. For the SELECT command alone, we use a different function 'rawQuery(String statement,String[] selectionArgs)'

The result of this statement will be a couple of rows put together, which is a portion of the entire table.

Cursor c=db.rawQuery("SELECT statement");

Functions in class Cursor:

moveToFirst()

moveToLast()

moveToNext()

move(int pos)

isFirst()

isLast()

isAfterLast()

Another set of Functions in class Cursor:

getString(int)

getInt(int)

getFloat(int)

getDouble(int)

getLong(int)

getCount()

After you are done with retrieving your data from a Cursor instance, it is important to 'close' the cursor connection.

c.close();

6. Closing a db Connection

Once all the work with a databse is over, it is necessary to close the databse connection.

db.close();
